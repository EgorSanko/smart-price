## ПРАВКА 1 · Слайд 2 «Актуальность»

### Проблема

В блоке «JBL FLIP 7 — один товар, пять цен» есть строка:

> `market.yandex.by` · 349,00 BYN · БЕЛАРУСЬ · MARKETPLACE

Яндекс.Маркет — российский маркетплейс. Несмотря на наличие `.by`-домена, визуально на слайде это создаёт диссонанс «Беларусь / Marketplace» и может вызвать вопрос комиссии.

### Что сделать

Заменить строку `market.yandex.by` на белорусский ритейл:

> `21vek.by` · 359,00 BYN · ↑ ДОРОЖЕ НА 5% · БЕЛАРУСЬ · РИТЕЙЛ

- Цвет точки/бейджа источника — нейтральный синий `#2196f3` (или серый `#8888a0`, если нужен «вторичный» оттенок)
- Стрелка вверх (↑) + текст «ДОРОЖЕ НА 5%» — оранжевым/жёлтым, как у других «дороже» строк
- Бейдж региона/типа — `БЕЛАРУСЬ · РИТЕЙЛ`

Остальные 4 строки (Onliner, Регард, World Devices, BigGeek) — без изменений.

## ПРАВКА 2 · Слайд 9 «Killer Feature — Найти дешевле»

### Проблема

На слайд добавили скриншот реального экрана `/cheaper` (справа) — это правильно, выглядит убедительно. Но левая половина слайда теперь скомкана: три карточки шагов, три карточки метрик и футноут — всё впихнуто в узкую колонку.

### Целевая структура

**Layout слайда 9 — двухколоночный 50/50:**

```
┌─────────────────────────────────────────────────────────────────┐
│  SP Smart Price                            ● 09 · KILLER FEATURE │
│                                                                  │
│  «Найти дешевле»: реверс WebSocket Яндекс.Алисы                  │
│  Товар прогоняется через бэкенд офферов Алисы. Парсим           │
│  payload'ы EAliceOfferCard / EAliceOffer / EAliceOfferList —    │
│  находим тот же товар у мелких продавцов на 10–40% дешевле.     │
│                                                                  │
│  ┌─────────────────────────┐   ┌─────────────────────────────┐ │
│  │  ЛЕВАЯ КОЛОНКА          │   │  ПРАВАЯ КОЛОНКА             │ │
│  │  (концепция, 3 шага)    │   │  (живой скриншот /cheaper)  │ │
│  │                         │   │                             │ │
│  │  3 элегантных блока     │   │  [НЕ ТРОГАТЬ]              │ │
│  │  с воздухом между       │   │  Реальный экран из прода    │ │
│  │                         │   │  с iPhone 17 Pro 119 090 ₽ │ │
│  │                         │   │  и top-3 предложениями      │ │
│  └─────────────────────────┘   └─────────────────────────────┘ │
│                                                                  │
│  ── ТИПОВАЯ ЭКОНОМИЯ 10–40% ── АНАЛОГОВ НА РЫНКЕ нет ──         │
│  ── ОСНОВАНО НА reverse-engineering WS ──                        │
│                                                                  │
│  FOOTNOTE · Исследовательский reverse-engineering протокола…    │
│                                                  09 / 14         │
└─────────────────────────────────────────────────────────────────┘
```

### Что сделать в левой колонке

**Убрать** текущие три мелкие карточки метрик (`10–40%`, `нет`, `reverse WS`) — они переезжают вниз слайда в виде моноширинной ленты под обеими колонками.

**Оставить и перерисовать** три шага. Каждый шаг — отдельный блок с **большим воздухом между ними** (gap ≥ 32 px), без рамок-карточек или с очень тонкой рамкой `rgba(255,255,255,0.06)`. Структура каждого шага:

#### Шаг 01

- **Лейбл** (моноширинно, мелко, серый): `01 · OPEN WS`
- **Заголовок** (sans-serif, 22 px, белый, bold): Открываем WebSocket
- **Описание** (regular, 14 px, приглушённый `#8888a0`): Устанавливаем сессию с бэкендом Алисы, эмулируем браузерный клиент устройства, выполняем handshake с UUID.
- **Код-бейдж** (моноширинно, маленький): `ws://...yandex.net/alice`

#### Шаг 02

- **Лейбл:** `02 · PARSE PAYLOAD`
- **Заголовок:** Парсим бинарный payload
- **Описание:** Извлекаем типизированные сущности из бинарного стрима — карточки офферов с ценой и продавцом.
- **Код-бейджи (3 шт. в строку):** `EAliceOfferCard` `EAliceOffer` `EAliceOfferList`

#### Шаг 03

- **Лейбл:** `03 · MERGE`
- **Заголовок:** Мерджим в общую ленту
- **Описание:** Нормализуем, сопоставляем с основным товаром по артикулу / EAN, добавляем в общий список выдачи.
- **Код-бейдж (success-зелёный фон `rgba(34,197,94,0.12)`, текст `#22c55e`):** `result: −10..40% vs Onliner`

### Что сделать в правой колонке

**Ничего не трогать.** Текущий скриншот `/cheaper` страницы с iPhone 17 Pro оставить как есть — это сильный визуальный аргумент. Только убедиться, что:

- Скриншот занимает **всю высоту левой колонки шагов** (≈ 480 px)
- Между скриншотом и шагами слева — gap **≥ 48 px**
- Скриншот выровнен по верху с первым шагом

### Что сделать внизу слайда (под обеими колонками)

Перенести метрики, которые раньше были левой колонкой, в **горизонтальную моноширинную ленту** во всю ширину слайда — между основным контентом и футером. Это разгрузит левую колонку:

```
─── ТИПОВАЯ ЭКОНОМИЯ 10–40%  ·  АНАЛОГОВ НА РЫНКЕ нет  ·  ОСНОВАНО НА reverse-engineering WS ───
```

- Моноширинный шрифт 11 px, uppercase, letter-spacing +0.1em, цвет приглушённый `#8888a0`
- Горизонтальные линии-разделители слева и справа от строки `———————`

### Футноут

Оставить **единой тонкой строкой в самом низу слайда**, под номером страницы:

`FOOTNOTE · Исследовательский reverse-engineering протокола в рамках академической работы. Не нарушает условий использования открытого API — работаем с трафиком собственного браузерного клиента.`

- 10 px, моноширинно, `#5a5a70`
- Не больше одной строки
